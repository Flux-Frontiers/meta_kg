refactor(cli): split monolithic cli.py into cli/ package with per-command modules

The 642-line `src/metakg/cli.py` has been replaced by a proper `src/metakg/cli/`
package where each command group lives in its own module.  No user-visible
behaviour changes; all existing entry-point names are preserved.

## Structure

```
src/metakg/cli/
├── __init__.py       — re-exports cli group + all entry-point aliases
├── main.py           — root @click.group() "metakg" with --version
├── options.py        — shared reusable Click option decorators
├── _utils.py         — shared helpers (timestamped filenames, arg parsers, output writer)
├── cmd_analyze.py    — metakg analyze / metakg analyze-basic
├── cmd_build.py      — metakg build / metakg enrich
├── cmd_mcp.py        — metakg mcp
├── cmd_simulate.py   — metakg simulate {fba,ode,whatif,seed}
├── cmd_viz.py        — metakg viz
└── cmd_viz3d.py      — metakg viz3d
```

## pyproject.toml changes

- `mcp` promoted from optional `[mcp]` extra to **core dependency** — the MCP
  server is now always available without any extra install flags
- `param` optional dependency removed (no longer needed)
- `[mcp]` extra removed; `[viz3d]` and `[all]` extras updated accordingly
- New `metakg` top-level script entry point added (`metakg.cli.main:cli`) so
  `metakg <subcommand>` works as a unified CLI alongside the existing
  `metakg-build`, `metakg-analyze`, etc. aliases

## mcp_tools.py

- Removed defensive `try/except ImportError` guard around `FastMCP` import
  (mcp is now a core dep, so the guard is unnecessary)

## docs/

- `docs/INSTALL.md` — new comprehensive step-by-step installation guide
  covering all extras, pathway download, build, enrich, kinetics seeding,
  MCP server, web explorer, 3D visualizer, dev install, env vars, and
  troubleshooting
- `docs/MCP.md` — new guide for integrating the CodeKG MCP server with
  Claude Code, Claude Desktop, GitHub Copilot, and Cline
- `docs/CAPABILITIES.md` — updated dependency tables: `mcp` moved to core,
  `[mcp]` extra removed, `[viz3d]` pinned versions added, install examples
  updated to reflect new extras layout; link to INSTALL.md added

## scripts/
